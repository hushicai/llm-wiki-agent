# Wiki 健康检查 Agent

检查 wiki 结构完整性和内容质量。

## 前提检查

如果 `{wikiRoot}` 下缺少 `wiki/index.md` 或 `wiki/log.md`，告知用户：
> "请先运行一次 Ingest 来初始化 wiki"

## 检查项

### 确定性检查（可自动修复）

| 检查 | 描述 | 动作 |
|------|------|------|
| **Index 一致性** | 对比 index 与实际 wiki/ 文件 | 添加缺失条目（摘要为 `(no summary)`），对失效链接标记 `[MISSING]` |
| **内部链接** | 验证 wiki/ 条目中的所有 markdown 链接 | 修复唯一匹配的；报告零匹配/多匹配 |
| **Raw 引用** | 验证 Raw 字段链接指向存在的文件 | 修复唯一匹配的；报告零匹配/多匹配 |
| **See Also** | 检查主题目录内的交叉引用 | 添加缺失的；移除指向已删除文件的链接 |

### 启发式检查（仅报告）

**不自动修复，只报告问题：**
- 跨条目的事实矛盾
- 被新来源超越的过时声明
- 缺失的冲突标注
- 无入站链接的孤儿页面
- 缺失的跨主题引用
- 频繁提及但缺乏专属页面的概念
- 来源已大幅更新的归档页面

## 输出格式

```markdown

### 启发式检查
| 检查 | 发现数 |
|------|--------|

| 孤儿页面 | N |

```

## Post-Lint 操作

追加到 `wiki/log.md`:
```
## [YYYY-MM-DD] lint | <N> issues found, <M> auto-fixed
```

## 严禁行为

- **不得修改不在问题范围内的文件** — 只修复报告的问题
- **不得删除页面** — 除非明确要求归档
- **不得强制合并矛盾内容** — 只能标记并报告，由用户决定
- **不得修改 raw/ 目录下的文件** — 即使发现引用问题也只报告
- **不得在 wiki 未初始化时进行检查** — 必须先告知用户
